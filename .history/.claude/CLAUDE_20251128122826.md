# グローバル指示

## コミュニケーション
- すべての応答は日本語で行うこと
- 技術的な説明も日本語を使用すること
- 変更理由を必ず説明すること
- 不明点があれば実装前に確認すること
- 関数の引数，引数の具体例，返り値などの構造を説明
- 条件分岐を視覚的に解説
- データフローを視覚的に解説


  ## 技術スタック
  - **フレームワーク**: React 19.1.1
  - **言語**: JavaScript (JSX)
  - **スタイリング**: SCSS Modules (`*.module.scss`)
  - **状態管理**: React Hooks (useState, useLocalStorage等のカスタムフック)
  - **ドラッグ&ドロップ**: @dnd-kit
  - **アニメーション**: Framer Motion
  - **ビルドツール**: react-scripts (Create React App)
  - **テスト**: Jest + React Testing Library
  - **ユニークID生成**: uuid (v4)

    ## コーディング規約

  ### コンポーネント設計
  - **関数コンポーネントのみ使用**すること（クラスコンポーネント禁止）
  - コンポーネントは `function ComponentName() {}` の形式で定義
  - propsは必ず分割代入で受け取る
  - デフォルトエクスポートを使用: `export default ComponentName;`

  ### ファイル命名規則
  - コンポーネント: PascalCase (`Header.jsx`, `SortableTask.jsx`)
  - カスタムフック: camelCaseで`use`プレフィックス (`useTheme.jsx`, `useDragAndDrop.jsx`)
  - スタイル: `*.module.scss` 形式

  ### 状態管理とロジック分離
  - **重要**: ビジネスロジックは必ずカスタムフックに分離すること
  - コンポーネントはUIの表示とイベントハンドリングのみに専念
  - 複雑な状態管理は `hooks/` ディレクトリに切り出す
  - LocalStorage連携は `useLocalStorage` フックを使用

  ### その他の規約
  - 配列操作では必ずイミュータブルな方法を使用（スプレッド演算子、filter、map等）
  - コンソールログは開発時のみ使用、本番コードには残さない

  ## 禁止事項

  ### 絶対に避けるべきこと
  - ❌ クラスコンポーネントの使用
  - ❌ グローバルスタイル（CSS Modules以外）の使用
  - ❌ `style.css` など通常のCSSファイルの新規作成
  - ❌ TypeScriptへの移行（現在のプロジェクトはJSXで統一）
  - ❌ Redux等の外部状態管理ライブラリの導入
  - ❌ インラインスタイルの乱用（既存コードでも最小限に抑える）
  - ❌ 直接的なDOM操作（`document.getElementById`等）
